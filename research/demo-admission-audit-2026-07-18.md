## 结论

- 全历史共审计 **315** 个候选：39 个具有可核验真实预览并完成六维人工评分，276 个因没有真实预览而正式准入分为 0。
- 最终 **26 个入选**，包括上一轮保留的 15 项和本轮完成真实 Demo 后通过的 11 项。
- 合计 **289 个拒绝**：276 个缺少真实预览，另有 13 个虽有可核验预览，但总分或核心维度没有达到发布门槛。
- 本轮研究来源规模为 134：100 个 AI Native 官网 + 34 个官方 GitHub 仓库。来源数量用于保证研究输入的多样性，不等于候选数量，更不等于发布数量。
- 本轮从这些来源形成的 73 个扩展候选，账目为 11 个 `current-build` + 43 个 `future-batch` + 19 个 `preflight-reject`。只有 11 项完成了本轮真实 Demo 与评分；其余 62 项没有制作真实预览，正式发布分仍为 0。

> 制作前预测分只能用于决定先做什么，不能进入发布账目。`future-batch` 即使预测 90 分，只要没有可运行 Demo、真实 GIF 与 provenance，六个发布维度都按 0 分处理。

## 产品判断

用户来到这个目录，是因为有艺术创作意图，却不知道怎样准确描述某种视觉或交互效果。因此发布实体必须是：

1. 不看标题也能辨认主体行为；
2. 能用稳定的中英文名称描述；
3. 能启发迁移到不同品牌、内容或产品；
4. 有真实运行结果、源码、捕获与来源证据；
5. 与目录中已有效果在主体行为上足够不同。

仓库 Stars、公司知名度、框架热度、研究来源数量和分类完整度都不能代替视觉质量。

## 同一套 100 分评分政策

本轮没有为扩展候选降低标准，仍使用 `2026-07-17` 版本政策。

| 维度 | 满分 | 高分要求 |
| --- | ---: | --- |
| 创意与差异性 | 20 | 有独立效果签名，不是通用组件换皮或既有 Demo 的装饰性变体 |
| 艺术与视觉完成度 | 20 | 构图、材质、色彩、层次、排版共同服务核心效果，缩略图仍成立 |
| 动效与交互编排 | 20 | 触发、时序、轨迹、缓动、反馈、循环或复位自然且因果清楚 |
| 效果辨识与可描述性 | 15 | 不依赖标题即可辨认，并能形成用户可复述的创作词汇 |
| 创作启发与迁移价值 | 15 | 能迁移到多个创作场景，不只证明某个框架 API 可运行 |
| 证据与可复现性 | 10 | 真实预览、实现来源、源码、版本、capture 与 GIF 一一对应 |

### 硬门槛

以下条件必须同时成立：

1. 先有可核验官方素材，或仓库内可运行 Demo 的真实浏览器录制；否则六维正式分直接为 0。
2. 总分至少 **80/100**。
3. 艺术与视觉完成度至少 **14/20**。
4. 动效与交互编排至少 **14/20**。
5. 效果辨识与可描述性至少 **11/15**。
6. 证据与可复现性至少 **8/10**。
7. 与已入选效果主体行为高度重复时，只保留证据、视觉完成度和迁移价值更强的一项。

## 最终发布集

最终发布目录只包含 26 个具有真实预览并通过同一门槛的条目：上一轮 15 项 + 本轮 11 项。所有被拒候选只能保留在带日期的研究审计中，不能继续留在 `effects.js`、README 表格或在线目录冒充 Demo。

后续任何新增或变更继续执行：证据预检 → 运行态与四帧检查 → 六维评分 → 相邻项并排去重 → 自动验证 → 发布。分类可以为空，发布门槛不能因数量目标而改变。
