# Parcel impact calibration textures

These three original raster assets are local, fictional packaging surfaces for `collision-reactive-3d-physics-stack`. They are not photographs of real brands, products, certifications, or logistics records.

## AI disclosure and use

- Generated on 2026-07-21 with Codex's built-in ImageGen tool (`product-mockup` use case).
- Each texture was generated independently. No user image, brand reference, trademark, remote runtime asset, or third-party raster was used.
- The generated 1254 × 1254 PNG sources were visually inspected for pseudo-text, logos, watermarks, malformed geometry, and thumbnail legibility. The three selected sources contain no textual claims.
- Project files were deterministically resized to 512 × 512 baseline JPEG at quality 84 with macOS `sips`; the generated originals remain in the local Codex image store and are not runtime dependencies.
- The p5 WebGL demo fetches and decodes every committed JPEG, verifies same-origin dimensions/byte checksums at runtime, transfers those exact decoded pixels into a `p5.Image`, and passes that image to `p5.texture()` for the boxes. The assets are therefore mechanism inputs, not a decorative background.
- Material colors, mass, restitution, friction, and handling limits are fictional test-fixture values chosen to make collision consequences readable. They must not be interpreted as real packaging specifications.
